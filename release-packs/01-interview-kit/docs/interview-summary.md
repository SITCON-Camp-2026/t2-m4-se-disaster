# Main Agent 訪談彙整

> 這份文件可以由 Codex 先產生草稿，但你必須刪除、修改或補充。請保留人類確認痕跡。

## 一句話結論

v1 先服務「會採取下一步的人」。它不是派工工具，而是把可閱讀的行動資訊和未確認報告分開，讓人知道自己正在看的是哪一種資料。

## 我們看到的問題

- 候選草稿太像任務時，行動者可能會照做。
- AI 寫得太順時，未確認資訊會看起來像事實。
- 只寫「需要確認」不夠，使用者需要知道要確認什麼。
- 「已確認」也不夠清楚，可能只確認了時間、地點、需求或其中一個欄位。

## v1 要幫誰

主要使用者是行動者，例如現場志工、小隊長、新手志工、運送協助者，以及會把資訊交給行動者確認的人。

他們最需要的不是更多整理結果，而是更快看出：

- 這不是正式任務。
- 這筆資訊在哪個分頁、風險程度如何。
- 缺少哪些確認。
- 如果要採取下一步，哪些判斷必須由人負責。

## v1 設計方向

- 把「原始資訊」「候選草稿」「確認問題」「正式任務」分清楚。
- 主畫面保留給行動者比較容易閱讀的資訊。
- 另設「未確認 / 高風險」分頁，放不能直接判斷、但行動者可能仍需要看到的報告。
- 在未確認分頁放明顯警示：系統沒有確認，也沒有建議出發或派工。
- 保留原文入口，讓使用者能回頭看原始資訊。
- 對 AI 產生或改寫的內容加明顯提醒。

## 明確不做

- 不判斷可以出發或不可以出發。
- 不派工。
- 不排救災優先順序。
- 不把候選草稿變成正式任務。
- 不用 AI 判斷資訊是否為真。
- 不因為報告看起來急，就自動視為可信或可行動。

## 還沒決定

- 行動者畫面要顯示多少原文。
- 局部確認先用文字紀錄，還是要做成欄位。
- 主畫面和未確認分頁之間，要由人手動分類，還是先依現有狀態分組。
